# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？
- リモートリポジトリはGitHub上に保管されているディレクトリ。チームで共有できる。
- ローカルリポジトリはPCに保管されているディレクトリ。


## プッシュとマージの違いは何でしょうか？
- プッシュはローカルリポジトリのファイルをリモートリポジトリに統合すること。
- マージは異なるブランチのファイルを統合すること。


## コミットとプッシュの違い
- コミットはaddで定めた範囲のファイルの変更履歴をローカルリポジトリ上で残す行為。
- プッシュはコミットした内容をリモートリポジトリ上に残す行為。チームに共有される。

## コミットのメッセージはどのように書いてあげるのが最適でしょうか？
- どんな変更を行ったか一目でわかるように短い文章で簡潔に書く。
- 各チームやプロジェクトのルールに則って書く。


## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？
- ローカルはターミナル上のコマンドのみで実行。
- プルリクエストはターミナル上のコマンド　→　GitHub上でプルリクの作成が必要（マージはリーダーやレビュアーが行う）。


## コンフリクトを起こしてしまった場合、どう対処すべきですか？
- 先にマージされた変更内容を取り込む。
- 後にマージしようとしている変更内容を取り込む。
- コンフリクトしたファイルを作成した人と責任者（PJ管理者やリーダー）と相談したうえでどちらの変更内容も取り込む。
